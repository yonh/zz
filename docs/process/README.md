# 流程文档

该目录包含跨任务的流程文档，用于说明工作如何在状态化文档体系中流转，以及如何配合 Windsurf workflow 使用。

## 文档列表
- [task-lifecycle.md](./task-lifecycle.md) - 任务如何在 `active-work`、`roadmap`、`reference`、`completed` 之间流转
- [windsurf-workflows.md](./windsurf-workflows.md) - 推荐的 Windsurf slash command 使用顺序
- [delivery-review-checklist.md](./delivery-review-checklist.md) - 实现完成后的独立审核清单
- [document-frontmatter-standard.md](./document-frontmatter-standard.md) - 任务文档 frontmatter 标准
- [frontmatter-validation-rules.md](./frontmatter-validation-rules.md) - 文档头校验与标准化规则
- [agent-skills.md](./agent-skills.md) - 哪些能力应作为隐式技能，哪些应作为显式 workflow
- [quick-start.md](./quick-start.md) - 如何快速开始一个新需求
- [templates/](./templates/) - 可复用任务模板

## 使用场景
当团队需要决定以下问题时，应先看这里：
- 文档应该放在哪里
- 下一条 workflow 命令是什么
- 什么时候允许归档
- 如何避免把当前交付和未来规划混在一起
- 如何统一审核标准
- agent 应默认执行哪些检查
