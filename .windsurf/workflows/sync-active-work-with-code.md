# /sync-active-work-with-code

目标：在开发前、开发中、开发后，检查 `docs/active-work/<task>/` 与当前代码实现是否仍然一致，避免文档和代码分叉。

执行步骤：

1. 读取目标任务目录：
   - `README.md`
   - `prd.md`
   - `tech-spec.md`
   - `implementation-plan.md`

2. 从 active-work 文档中提取四类信息：
   - 当前目标
   - 当前范围内承诺的实现项
   - 当前验收标准
   - 明确排除项

3. 在代码中核对这些承诺是否真实存在：
   - 查找实际入口文件和核心逻辑
   - 确认文档提到的字段、结构、接口、配置是否已经存在
   - 标记“文档已写但代码未落地”的项
   - 标记“代码已变化但文档未更新”的项

4. 特别检查以下偏差：
   - Active 文档是否混入 roadmap 内容
   - 文档是否引用不存在的文件、结构、接口、配置项
   - 文档是否把未来态示例写成当前态能力
   - 文档是否遗漏已经实现的重要行为变化

5. 输出结果时按三类分类：
   - **Aligned**：文档与代码一致
   - **Missing in Code**：文档承诺但代码未实现
   - **Missing in Docs**：代码已变化但文档未同步

6. 如果发现 Active 文档已不适合作为当前实现依据：
   - 先更新 Active 文档边界
   - 再继续开发或继续审核

7. 最终输出：
   - 当前是否允许继续按 active-work 开发
   - 必须先修的文档问题
   - 必须先修的实现问题
   - 是否需要把某些内容移回 roadmap
