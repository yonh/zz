# /breakdown-active-work

目标：把 `docs/active-work/<task>/` 中的任务拆成当前最小可执行批次，避免过度设计和大批量实现。

执行步骤：

1. 读取目标任务目录：
   - `README.md`
   - `prd.md`
   - `tech-spec.md`
   - `implementation-plan.md`

2. 检查代码现实，而不是只相信文档：
   - 查找真实代码入口
   - 确认现有能力已经覆盖了什么
   - 找出真正缺失的最小改动点

3. 把当前任务压缩成一个最小批次，优先满足：
   - 可在 1 天内完成
   - 改动面尽量小
   - 不引入新的架构层
   - 可明确验收

4. 对每个子任务输出：
   - 目标
   - 涉及文件
   - 实现边界
   - 验收标准
   - 回归检查

5. 使用 `todo_list` 建立当前批次计划：
   - 只能有一个 `in_progress`
   - 其余为 `pending`
   - 不把 roadmap 内容混入当前待办

6. 如果发现 Active 文档仍夹带未来工作：
   - 停止直接实现
   - 先回写文档边界
   - 再重新拆分任务

7. 结束时输出：
   - 当前批次任务列表
   - 推荐实施顺序
   - 明确不做的内容
